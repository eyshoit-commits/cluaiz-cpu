# 🤝 Contributing to Cluaiz

First of all, thank you for your interest in contributing to Cluaiz! By contributing, you are helping build a high-performance, sovereign neural infrastructure.

## 🏛️ The Contributor Commercial Waiver (Tier B)
As defined in the **Cluaiz Systems License (CSL) v1.0**, significant contributors are granted a **Full Commercial Waiver**. 

- **How to Earn it**: Once your Pull Requests (PRs) providing significant architectural improvements or core kernels are merged into the official repository, you will be granted a permanent commercial license bypassing the $1M revenue threshold.
- **Why?**: We believe those who help build the core should own the right to profit from it.

## 🛠️ Development Standards
- **Language**: All core logic must be written in **Rust** (for safety/speed) or **C++/CUDA** (for kernels).
- **Architecture**: Every contribution must adhere to the **3-Tier Silicon-First** design. No monolithic bloat.
- **Testing**: PRs without verification tests or performance benchmarks will not be merged.

## 🚀 How to Submit a PR
1. Fork the repository.
2. Create a feature branch: `git checkout -b feat/your-feature`.
3. Commit your changes with descriptive messages: `git commit -m "feat: implement ternary kernel optimization"`.
4. Push to your fork and open a Pull Request against the `main` branch of `cluaiz/cluaiz`.
5. Mention how this contribution helps the Cluaiz ecosystem in your PR description.
