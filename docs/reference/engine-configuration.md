# Cluaiz Unified Configuration Reference Manual

This manual details all configurable parameters used inside Cluaiz JSON settings files stored in `~/.cluaiz/engine/`. It consolidates the configuration tables for `system_control.json`, `system_booster.json`, and `Permission.json`.


---

## ⚙️ 1. Silicon Truth & Hardware Governance Profile (`system_control.json`)

### Architectural Role

The system control profile is generated by the engine's hardware prober at startup. It registers host environment constraints (AVX capabilities, CPU cores, physical GPU memory) to ensure that runtime executions do not cause system lockups or memory thrashing.

For a deeper dive into the system control implementation and dynamic hardware probing system, see the [System Control Concept Document](../engine/system_control.md).

### Configuration Schema Table

| Parameter | Type | Default | Options / Valid Ranges | Description |
|:---|:---|:---|:---|:---|
| **`node_id`** | String | `cluaiz-node-x1` | Alphanumeric | Unique workspace identifier for local system discovery. |
| **`active_model`** | String | `bonsai:8b` | Valid Model ID | Active model loaded in the server session. Mapped from `CoreRoster` lists. |
| **`user_identity.name`** | String | `Operator` | Text | Profile name of the system operator. |
| **`user_identity.purpose`** | String | `PRODUCTION` | `RESEARCH` / `PRODUCTION` | Standard mode optimizations. `RESEARCH` favors precision, `PRODUCTION` maximizes tokens/sec. |
| **`hardware_governance.vram_limit_gb`** | Float | `12.0` | `0.1` to physical GPU max | Hard maximum ceiling for VRAM allocations to protect host OS. |
| **`hardware_governance.cpu_thread_limit`** | Integer | `8` | `1` to logical thread count | Maximum thread pool concurrency allocated for BLAS/matrix math operations. |
| **`hardware_governance.allow_speculative_decoding`** | Boolean | `true` | `true` / `false` | Global driver permission for draft model execution. |
| **`hardware_governance.fallback_to_cpu`** | Boolean | `true` | `true` / `false` | Dynamically executes operations on CPU if GPU registers OOM. |
| **`network.api_host`** | String | `127.0.0.1` | Loopback / Local IP | Binds Axum HTTP engine interfaces. |
| **`network.api_port`** | Integer | `3000` | Port Range | Network port allocation. |
| **`network.enable_cors`** | Boolean | `true` | `true` / `false` | Restricts access to localhost clients (desktop/mobile app) to prevent host-data exfiltration. |

## 🚀 2. Compute Acceleration & Optimization Profile (`system_booster.json`)

### Architectural Role

The system booster profile controls the lowest-level model compilation limits, layer allocation splits, and memory caching optimizations. It is processed directly by the VRAM Arbiter in the Master Router to govern memory allocation boundaries and balance load between Llama and ONNX backends.

For exhaustive parameter implications and physical hardware consequences, see the [System Booster Explanation Guide](../engine/booster.md).


### Configuration Schema Table

| Parameter | Type | Default | Options / Valid Ranges | Description |
|:---|:---|:---|:---|:---|
| **`mode_run`** | String | `balance` | `edge` / `multitasking` / `balance` / `max_boost` / `ultra_max_boost` / `hyper_cluster` | Determines CPU thread scheduler priority settings. `max_boost` locks threads at real-time priority. |
| **`n_gpu_layers`** | Integer | `-1` | `-1` to `128` | Number of transformer layers offloaded to GPU VRAM (`-1` = full offload, `0` = CPU only). |
| **`kv_quant`** | String | `auto` | `auto` / `kv16` / `kv8` / `kv4` | Compression level for intermediate Key-Value (KV) cache context. |
| **`context_shift`** | String | `auto` | `auto` / `off` / `minimal` / `standard` / `aggressive` / `extreme` | Rolling KV cache compression strategy. Drops old context while preserving system prompts. |
| **`flash_attention`** | String | `auto` | `on` / `off` / `auto` | Toggles Flash Attention SRAM tiling optimizations. |
| **`speculative_decoding`** | String | `auto` | `on` / `off` / `auto` | Toggles Draft-model speculative decoding (requires compatible models). |
| **`turbo_quant`** | String | `auto` | `on` / `off` / `auto` | Applies Givens rotations to mathematically minimize quantization losses. |
| **`auto_round`** | String | `auto` | `on` / `off` / `auto` | Engages dynamic rounding during matrix multiplications. |
| **`dflash`** | String | `auto` | `on` / `off` / `auto` | Dynamic Flash Attention - adjusts memory footprint frame-by-frame. |
| **`think_mode`** | String | `auto` | `on` / `off` / `auto` | Forces internal hidden Chain of Thought (CoT) reasoning phase. |
| **`response_length`** | String | `auto` | `auto` / `short` / `long` / Integer | Sets hard limit boundaries on token response lengths. |
| **`enforce_json`** | Boolean | `false` | `true` / `false` | Logit processor mask to enforce strictly valid JSON structure. |
| **`force_memory_lock`** | String | `off` | `on` / `off` / `auto` | Invokes `mlock` / `VirtualLock` to prevent OS memory swapping. |
| **`force_vram_reclaim`** | String | `off` | `on` / `off` / `auto` | Disables memory pooling to free every tensor instantly. |

## 🔒 3. Security Boundaries & Capability Rules Profile (`Permission.json`)

### Architectural Role

The permission profile serves as the primary gateway authorization file, determining sandbox execution limits, telemetry access, and model allocations. During execution of dynamic WebAssembly components, this profile governs the OS syscall filter ring.

For detailed information on JIT isolation and how sandbox execution loops operate, see the [Permissions Architecture Manual](../engine/permission.md) and the [JIT Injection Architecture Explanation](../engine/jit_architecture.md).


### Configuration Schema Table

| Parameter | Type | Default | Options / Valid Ranges | Description |
|:---|:---|:---|:---|:---|
| **`firewall_mode`** | String | `strict` | `auto` / `strict` / `off` | Sandbox execution ring limits for dynamic WASM components. |
| **`enable_telemetry`** | Boolean | `false` | `true` / `false` | Opt-in flag to send performance telemetry metrics. |
| **`vectorize_user_input`** | Boolean | `true` | `true` / `false` | Automatically embeds and logs prompts into local vector database. |
| **`vectorize_ai_response`** | Boolean | `true` | `true` / `false` | Automatically embeds and logs AI replies. |
| **`chat_ttl_hours`** | Integer | `24` | `12` to `8760` | Chat database TTL expiration interval. |
| **`default_chat_model`** | String | `bonsai:8b` | Valid Model ID | Selected default model for conversational interface threads. |
| **`default_vector_model`** | String | `bge_m3` | Valid Model ID | Selected default embedding model. |
