### 4. 🧪 MATHEMATICAL AUDIT: TERNARY ADDITION (BITNET)
BitNet models (Bonsai) require a custom activation function: **RMSNorm without a Gain Parameter**.

#### 4.1 Activation Trace
Industry standard for BitNet 1.58b:
`Active(X) = (X - Mean(X)) / StdDev(X)`
- **Observation**: Standard Llama uses `RMS(X) * Gain`. 
- **The Failure**: If we use the standard Llama kernel on Bonsai, the "Gain" (which is missing/null in BitNet) is treated as a 0 weight, killing all activations to zero. **Bonsai is mathematically "muted" in Archer Native.**
