PART 2: INTERNAL FILE SPECIFICATIONS
в”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓв”Ѓ

Each sub-skill folder MUST strictly contain:

1.  **manifest.json**
    - Metadata for the `SkillRegistry`.
    - Triggers used by the `router.bin`.
    - Permission levels (ReadOnly, WorkspaceWrite, SystemAccess).

2.  **soul.atma**
    - The binary neural weight delta.
    - Can be a KV-Cache prefix or an Activation Steering Vector.
    - Compressed format for fast MMAP loading.

3.  **logic.wasm** (Optional)
    - Compiled WebAssembly binary.
    - Contains the deterministic part of the skill (Calculations, API calls).
    - Ensures platform-independent, sandboxed execution.

4.  **connector.mcp** (Optional)
    - Configuration for external Model Context Protocol servers.
    - Defines how to connect to third-party databases or web services.
