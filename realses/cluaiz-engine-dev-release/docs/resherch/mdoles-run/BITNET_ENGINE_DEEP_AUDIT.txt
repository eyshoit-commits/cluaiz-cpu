### 1. 🔍 THE TERNARY REVOLUTION (THE 1-BIT TRUTH)
BitNet (specifically the 1.58b version used by Bonsai) is not a 1-bit model in the traditional sense; it uses **Ternary Values** {-1, 0, 1}. This requires a completely different mathematical approach compared to standard Llama models.

#### 1.1 BitLinear: The Core Primitive
Traditional Linear Layer: `Y = WX + b` (where W is FP16 or INT4).
BitLinear Layer: `Y = BitNetMultiply(W, X)`. 
- **The Magic**: The multiplication is replaced by simple additions and subtractions (since W is -1, 0, or 1).
- **Physical Reality**: This reduces memory bandwidth by 10x-20x but requires high-precision **Input Quantization** (RMSNorm + Scaling).

**CRITICAL GAP**: Archer's `SiliconOrchestrator` is trying to load Bonsai into Runtime B (Llama.cpp). Llama.cpp doesn't have a native "Ternary Adder" kernel. It tries to dequantize the 1-bit weights into FP16, but since the weights are stored in a specialized packed format, the result is "Mathematical Noise" (ERROR 404).
